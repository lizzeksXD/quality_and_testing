1. Зачем отключать аутентификацию (LOGIN_DISABLED = True) в тестах? В каких случаях
это допустимо?
Ответ: аутентификация может быть временно отключена (LOGIN_DISABLED = True) только если
тестируется логика CRUD, а не сама аутентификация.

2. Какие типы исключений SQLAlchemy вы ловите в тестах? Что они означают?
Ответ: IntegrityError - нарушение правил БД (дубликат, пустое поле, нет родительской записи)
DataError - неправильные данные (слишком длинный текст, не тот тип)
NoResultFound - ничего не найдено по запросу
MultipleResultsFound - нашлось несколько записей вместо одной
ProgrammingError - ошибка в SQL-запросе (опечатка, нет таблицы)
OperationalError - проблемы с подключением (БД упала, нет сети).

3. Почему важно использовать follow_redirects=True при тестировании POST-запросов?
Ответ: потому что POST-запросы обычно перенаправляют (redirect) после успешного выполнения, и без follow_redirects=True будет проверять не тот ответ.

4. Как можно улучшить тесты, чтобы они не отключали аутентификацию, а работали с
реальным входом?
Ответ: вместо отключения аутентификации можно воспроизвести реальный вход пользователя или создать авторизованные сессии.

5. Какие ещё граничные случаи можно протестировать для модели Book?
Ответ: Пустая строка, отрицательный год, HTML-инъекция, 0 страниц, цена -10, дата 31 февраля.
